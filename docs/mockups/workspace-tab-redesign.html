Workspace tab redesign

Side-by-side preview of the proposed workspace settings surface. Filters, navigation, and CRUD interactions simulate behavior in-memory — nothing persists, and no plugin code is wired. Use the left rail to step through sub-views; use the material toggle to compare surface treatments.

  • Excluded from this pass: workflow editor, file picker. Deferred to a second mockup if needed.
  • Not redesigned: workspace list view — already shipped via SearchableCardManager. The list above projects/tasks/states intentionally matches its visual language.
  • Mobile hint: a 390 px viewport preview is rendered alongside the desktop view. Icon-only buttons sized 40×40 px for ≥WCAG 2.5.5 tap targets.
  • Issue #217 coverage: F1 (empty-name validation) and F3 (in-flight disable on Archive/Restore/Delete) are demonstrable. F2 (no-op no-event) and F4 (stale-render guard) are invisible to a mockup — see the annotation card on the States view.
  • Material caveat: the project pivoted to faux-glass for production (no backdrop-filter on most workspace-tab chrome — Android FPS + Obsidian flat-theme constraints). Flat and Faux-glass map directly to what production will ship; Liquid glass is a visual-reference variant that uses backdrop-filter and will NOT ship as-is on workspace surfaces. The glass tokens (--glass-fill, --glass-rim, etc.) mirror styles.css names so the production migration is a name-swap, not a value-swap.

Snapshots of workspace context that can be resumed later. Embedded inline on the workspace detail view; this dedicated view shows the same component with extra breathing room and the issue #217 demo affordances.

Issue #217 coverage
  • F1 — empty-name validation: Try editing a state and clearing the name field, then press Save. The name input shows an inline error and the Save button stays disabled until non-whitespace text is entered.
  • F3 — in-flight disable on Archive / Restore / Delete: The action icons display a spinner and are disabled for ~700 ms after click in this mock. Production code will guard against the double-click race using the same disable-then-await pattern.
  • F2 — no-op save no-event (not demonstrable in mockup): When the user saves without changing any field, production code will skip the storage write so state_updated is not emitted. The mockup cannot show the absence of an event.
  • F4 — stale-render guard (not demonstrable in mockup): Rapid workspace switching can cause an older listStates promise to stomp a newer render. Production code will tag each in-flight load with a generation counter and discard resolved results that belong to a superseded generation.
Mobile (390 × 800) Same component, narrow viewport. Tap targets ≥ 40 × 40.

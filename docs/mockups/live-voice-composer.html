Nexus voice mode

Live composer takeover

This mock keeps text chat as the primary surface. Live voice only replaces the composer with a red session dot, a stateful waveform, and a stop button.

State
Design notes

The live composer itself intentionally has no visible text. The dot communicates that the session is active, the waveform communicates who is speaking, and the square button exits live mode.

Transcript content stays in the normal message stream so live voice feels like another way to drive the same chat.

Design Changes

Header: 36px → 36px (same height, denser)
  • Padding reduced from 12px 16px to 6px 10px
  • Title font 16px → 14px, left-aligned instead of centered
  • Header buttons packed tighter with gap: 2px
Message bubbles: ~40% less vertical space
  • Padding from 0.875rem 1rem → 0.5rem 0.75rem
  • Role icons removed — user/assistant distinguished by alignment + subtle color
  • Message header row eliminated (was icon + actions, now actions are tap-to-reveal)
  • Bubble border-radius from 1rem → 0.75rem
  • Container width 95% → 85% for user, full for assistant (chat-app pattern)
  • Gap between messages: 0.25rem → 0.375rem (tighter but readable)
Action buttons: tap-to-reveal overlay
  • Actions hidden by default (no opacity:0.85 always-visible)
  • Single tap on message reveals floating action pill
  • Tap outside or second tap dismisses
  • Saves ~28px per message (action row height eliminated)
Input: single-line default, auto-expands
  • min-height from 64px → 40px (single line)
  • Auto-grows as user types, up to 120px
  • Placeholder shortened: "Type your message..." → "Message..."
  • Container padding from 8px 12px → 6px 10px
  • Saves ~32px in default state
Context bar: collapsed to micro-bar
  • Full progress bar (60px) → 16px inline strip below input
  • Shows abbreviated info: "2k / 1M" and cost
  • Color-coded fill still visible as thin 3px bar
  • Tap to expand full context details (future)
  • Saves ~44px
Total vertical savings: ~90-120px
  • Header: ~6px saved
  • Per-message: ~20-30px saved (compounds with message count)
  • Input area: ~32px saved
  • Context bar: ~44px saved
  • On a 667px viewport with keyboard (280px visible), this means 2-3 more messages visible

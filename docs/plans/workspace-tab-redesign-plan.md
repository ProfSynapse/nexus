#### Design Approach

- **BoxedSection** is the load-bearing primitive — 6+ uses across detail + sub-pages. Extract to shared module. Imperative body callback (matches Obsidian renderer convention); declarative body API would just forward the same callback with extra indirection.
- **Row primitives** stay as `.setting-item` CSS modifiers (`.is-task`, `.is-project`, `.is-note`, `.is-dependency`). **No** `RowList<T>` abstraction — each row composes differently (status dots, link-type pill, action cluster), and a shared wrapper would just take a per-variant render callback.
- **WorkspacesView routing** stays as single enum + switch in `WorkspacesTab.render()`. 7 states is fine for explicit dispatch; sub-router primitive adds indirection without removing it.
- **WorkspaceDetailRenderer split** triggered if file exceeds 800 LoC after BoxedSection port. Extract `renderProjectDetail` → `ProjectDetailRenderer`, `renderTaskDetail` → `TaskDetailRenderer`. Public method names stable for callers.
- **Picker affordance** is the inline `.ws-empty` hint paragraph that auto-dismisses after 1.8s — **NOT** Obsidian `Notice` (different surface). Real task/note picker deferred to follow-up issue.

#### Key Decisions

| Decision | Options | Resolution | Rationale |
|----------|---------|------------|-----------|
| BoxedSection extraction | inline / shared | **Shared module** at `src/settings/components/BoxedSection.ts` | 6+ uses; sibling to BackButton, BreadcrumbNav. |
| ConfirmModal shape | 1 class + variant prop / N classes | **1 class, `variant: 'delete'\|'remove'\|'archive'`** | Variation is copy + CTA label; structural shape is identical. |
| Row primitives | `RowList<T>` / `.setting-item` CSS modifiers | **CSS modifiers** on `.setting-item` | Per-row variation too large for shared wrapper. |
| WorkspaceDetailRenderer split | god-renderer / per-page | **Per-page if >800 LoC after port** | Natural split axis is sub-page; protects file-size budget. |
| WorkspacesView routing | enum + switch / sub-router | **Enum + switch** (unchanged from today) | 7 explicit states; no deep-linking/back-stack needs. |
| Picker UX | real picker / inline hint | **Inline hint (deferred picker)** | "When in doubt, leave it out." Real picker is a follow-up issue. |
| StatesSection port | greenfield / shell-only | **Shell-only port** | Preserve PR #216 service contract + `Show archived` toggle. |
| Task row markup | `<table>` / flex `.setting-item` | **Flex `.setting-item.is-task`** | Mockup-aligned; loses table-column semantics but density gain compensates. |
| Form primitives | bare HTML / Obsidian framework | **Obsidian framework** (`DropdownComponent`, `TextComponent`, `Setting`) | Mockup used bare HTML as shortcut; production uses framework primitives. |

- `.task-meta` gap tightens
- `.ws-tree-row` depth indent caps via `clamp()` to prevent overflow
- `@supports (position: sticky)` for graceful degradation on older iOS WebView

#### Obsidian Framework Primitive Substitution

The mockup used bare HTML for forms (shortcut). Production should use:
- `DropdownComponent` instead of `<select class="ws-select">` (status filters, project picker, priority dropdowns)
- `TextComponent` instead of `<input class="ws-input">` (title, tags)
- `TextAreaComponent` instead of `<textarea class="ws-textarea">` (description)
- `ButtonComponent` for primary buttons (already in use)
- `Setting` for form-row labels where layout fits (Task detail, Workflow editor); custom `.ws-field` only where `Setting`'s mandatory layout doesn't fit (inline-icon variants)
- `Notice` for global toasts (save success, errors) — but **not** for the inline picker hint (different surface)
